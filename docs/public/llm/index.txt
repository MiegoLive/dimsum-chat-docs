# DimSum Chat AI Agent 开发指南 — 知识资源索引
# 版本：1.0.1 | 更新日期：2026-06-09

================================================================
阅读说明
================================================================

本目录为 DimSum Chat 点心应用开发的 AI Agent 静态知识库。
你（AI Agent）应在收到开发者"帮我开发一个点心应用"类请求时，
按照以下顺序读取文件：

  1. rules.txt          ← 必须先读！元规则引擎，包含决策树和强制检查清单
  2. shared.txt         ← 核心共享知识（SDK API、配置格式、消息类型、安全规范）
  3. parser-types.txt   ← Parser 完整类型声明！遇到任何 Parser 属性必须先查此文件
  4. 根据规则引擎选择的模式，读取对应文件：
     - mode-single.txt  → 单 HTML 模式（零前端基础）
     - mode-multi.txt   → HTML+JS+CSS 模式（有前端基础）
     - mode-template.txt → Vue3 模板模式（前端工程师）

================================================================
文件职责边界
================================================================

| 文件               | 职责                                          | 何时读取      |
|-------------------|----------------------------------------------|-------------|
| rules.txt         | 元规则：决策树、强制检查清单、AI 行为约束         | 必须首先读取   |
| shared.txt        | 共享知识：SDK API、配置格式、消息类型、安全       | 规则引擎之后   |
| parser-types.txt  | Parser 完整类型声明：所有属性的类型/平台/说明      | 写代码前必查   |
| mode-single.txt   | 单 HTML 模式完整工作流与代码模板                 | 选择模式一后   |
| mode-multi.txt    | HTML+JS+CSS 模式完整工作流与代码模板            | 选择模式二后   |
| mode-template.txt | Vue3 模板模式完整工作流                         | 选择模式三后   |

================================================================
AI Agent 行为准则
================================================================

1. 必须按照 rules.txt 中的决策树帮助开发者选择模式，不可跳过
2. 选择模式后，必须完整执行对应模式文件中的工作流
3. 每个模式中的强制检查清单项目不可跳过
4. 遇到安全相关操作（XSS 防护、配置验证）必须严格执行
5. 不要自行发挥——所有约定以本知识库文件为准
6. 如果开发者情况超出三种模式的覆盖范围，引导至人工支持
7. ⚠️ 遇到 Parser 的任何属性，必须查阅 parser-types.txt，禁止猜测字段名或类型
